En el modelo de caja el contenido sería el espacio que está ocupando el texto. Pero podría ser una imagen lo que ocupase ese espacio en vez de texto

Despues vendría el padding o relleno. El padding de un elemento es la cantidad de espacio entre el borde lo amarillo y el contenido del elemento las flechas en este caso . Se dan entre uno y cuatro valores, donde cada valor puede ser una longitud o un porcentaje. Los valores en porcentaje se refieren al ancho del elemento padre. No se permiten los valores negativos. Si se dan cuatro valores, se aplican a los rellenos superior, derecho, inferior e izquierdo, respectivamente. Si se da un valor, se aplica a todos los lados. Si se dan dos o tres valores, los valores faltantes se toman del lado opuesto.

El borde es una de las cuatro partes del modelo de caja en CSS. Es la parte que rodea el contenido y el padding de un elemento. La parte rosa en este caso. Con width se especifica el ancho de los bordes. Cada uno de los 4 bordes puede tener unas características diferentes con respecto a los demás (anchura, color, etc)

    style especifica el estilo de los bordes. Algunos de esos estilos son estos:
  • "solid" solido
  • "dashed" discontinuo
  • "dotted" punteado
  • "double" borde doble

    ¿Qué ha ocurrido? pues la explicación es que si no se usa la propiedad box sizing tanto el padding como el borde tendrán sus dimensiones propias y podrán agrandar el contenedor. Mientras que si se usa box sizing el padding y el borde pasan a estar dentro de el alto y el ancho que se le de al contenedor. con box sizing un contenedor con un ancho de 100px y un alto de 100px tendrá esas dimensiones mientras que sin box sizing habrá que sumarle además de el ancho y el alto lo que corresponda de padding y de borde
